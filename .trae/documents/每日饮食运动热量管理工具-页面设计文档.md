## Global Styles（全局样式与设计规范）

### Layout
- 桌面优先：内容容器最大宽度 1200px，居中；左右留白随屏幕扩大。
- 页面主栅格：12 列 CSS Grid（gap 16px），卡片内用 Flex 进行对齐。
- 响应式断点：
  - ≥1200px：三列信息卡/图表并排为主
  - 768–1199px：两列布局
  - <768px：单列堆叠（作为兼容，不作为主设计）

### Typography
- 字体：系统字体栈（优先苹方/思源黑体/系统默认）。
- 字号：
  - H1 24/32
  - H2 18/26
  - Body 14/22
  - Meta 12/18

### Color & Tokens
- Background: #0B1020（深色底）或 #F6F7FB（浅色底，默认）
- Surface（卡片底）：#FFFFFF
- Primary: #2563EB（按钮/强调）
- Success（缺口为正）：#16A34A
- Danger（超额）：#DC2626
- Border: #E5E7EB
- Shadow：轻量阴影（卡片层级）

### Buttons / Inputs
- 主按钮：Primary 实心；hover 加深 6–8%；disabled 降低不透明度。
- 次按钮：描边 + 透明底；hover 轻微底色。
- 输入：默认高度 40px；错误态红色描边并给出短错误文案。

### Data Display（数字与单位）
- kcal 数值默认取整；允许在编辑框输入小数并最终四舍五入。
- 关键指标展示顺序：摄入 → 消耗 → 净摄入 → 缺口/超额。

## Page 1：今日概览页（/）

### Meta Information
- Title：今日热量概览
- Description：查看当日饮食摄入、运动消耗与热量缺口。
- Open Graph：title 同上；type=website。

### Page Structure
- 顶部：应用导航栏（左：产品名；中：主导航；右：日期选择）。
- 主体：上方指标卡区 + 下方两栏（最近记录 / 目标设置）。

### Sections & Components
1. 顶部导航栏（Sticky）
   - 左侧 Logo/名称：点击回到今日。
   - 主导航 Tabs：今日概览 / 记录 / 报表。
   - 日期选择：
     - 快捷：今天、昨天
     - 日历弹层：选择任意日期
2. 关键指标卡片区（Grid）
   - 4 张 KPI 卡：摄入、消耗、净摄入、缺口/超额
   - 缺口卡的状态色：缺口为正（绿色）/超额（红色）
   - 卡片右上角：小号“查看报表”链接（跳转 /reports 并带日期参数）
3. 快捷操作区（在 KPI 下方）
   - 主按钮：拍照记饮食（跳 /log 并默认打开“饮食识别”面板）
   - 次按钮：手动记饮食（跳 /log 并聚焦饮食表单）
   - 次按钮：记录运动（跳 /log 并聚焦运动表单）
4. 最近记录摘要（左栏）
   - 列表分组：饮食 / 运动
   - 每条：名称 + 时间 + kcal
   - 行内操作：编辑、删除（删除二次确认）
5. 目标设置（右栏卡片）
   - 字段：每日目标 kcal（数字输入）
   - 保存策略：输入失焦自动保存 + 显示“已保存/保存失败”轻提示

### Interaction States
- 指标卡加载中：骨架屏；失败：错误条 + 重试按钮。
- 删除记录：弹窗确认，确认后列表与 KPI 立即刷新。

## Page 2：记录页（饮食&运动）（/log）

### Meta Information
- Title：记录饮食与运动
- Description：通过拍照识别食物，或手动记录饮食与运动。
- Open Graph：title 同上。

### Page Structure
- 左侧（主列）：分段卡片（饮食识别/饮食手动/运动记录）
- 右侧（辅列）：当日汇总与当日记录列表

### Sections & Components
1. 饮食图片识别卡片
   - 上传区：拖拽/点击上传（桌面优先），显示图片预览
   - 操作：开始识别（主按钮）
   - 识别结果：
     - 候选项列表（单选）：名称、估算 kcal、置信度（用条形/百分比）
     - “手动编辑”切换：进入可编辑表单
   - 异常：识别失败提示 + 允许重试或改为手动录入
2. 饮食确认与校正表单（与识别结果联动）
   - 字段：食物名称、份量（克/份）、热量（kcal）
   - 保存按钮：保存饮食记录
   - 校验：kcal 必填且>0；名称必填
3. 运动记录卡片
   - 字段：运动类型（文本/下拉）、时长 min、强度（低/中/高 可选）、体重 kg（可选）
   - 自动计算：输入变更后实时刷新“预计消耗 kcal”
   - 保存按钮：保存运动记录（允许手动改 kcal）
4. 右侧当日汇总卡片
   - 展示：摄入、消耗、净摄入、缺口/超额
   - 提示：与目标差值的颜色状态
5. 当日记录列表（右侧）
   - Tab：饮食 / 运动
   - 行内编辑：点击进入抽屉/弹层编辑（不离开页面）
   - 删除：二次确认

### Responsive Behavior
- ≥1200px：左右双栏
- 768–1199px：上下两段（右侧内容下移）

## Page 3：报表页（热量缺口）（/reports）

### Meta Information
- Title：热量缺口报表
- Description：按时间范围查看摄入、消耗与缺口趋势。
- Open Graph：title 同上。

### Page Structure
- 顶部：筛选条（时间范围、快捷区间）
- 主体：趋势图 + 汇总卡 + 明细表

### Sections & Components
1. 筛选条
   - 快捷区间：最近7天、最近30天、自定义
   - 自定义：起止日期选择器
   - 导出按钮：导出 CSV（当前筛选范围）
2. 趋势图（主图）
   - 折线：缺口/超额（deficitKcal）
   - 柱状/面积：摄入与消耗（可开关）
   - 交互：hover 显示 tooltip（日期、摄入、消耗、净摄入、目标、缺口）
3. 汇总卡片（图下方）
   - 指标：总摄入、总消耗、平均缺口、缺口为正天数/超额天数
4. 明细表（可钻取）
   - 列：日期、摄入、消耗、净摄入、目标、缺口
   - 行点击：打开当日明细弹层（含饮食/运动列表）
   - 弹层内：提供“去修改记录”按钮跳转 /log 并带日期参数

### States
- 空数据：展示“暂无记录”，并引导去记录页添加第一条记录。
- 导出中：按钮 loading；导出完成下载文件。
